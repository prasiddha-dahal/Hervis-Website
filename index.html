Hervis

Hervis is a plugin-oriented ambient computing layer for Linux — a gesture-driven, context-aware interface that sits between you and your desktop. It integrates voice, vision, LLMs, and window management into a single hot-swappable runtime.

I'm building Hervis because the next interface isn't a chatbot or an app — it's the OS itself becoming aware.

Excited about:

  • Gesture-native interaction: V for camera, palm-move for windows, pinch to resize — no keyboard required
  • Hot-swappable plugins: swap context engines, voice models, or vision backends at runtime without restarting
  • Ambient wake: Hervis listens, watches, and acts — idle until you need it, instant when you do

components

Hervis is built from three interlocking layers — each independently pluggable and hot-swappable.

Core
  • Tokenizer & Minimal Parser
  • Context Awareness Engine
  • Modes — command, idle, action
  • Voice Recognition
  • LLM Integration (llama-class tiny models)
Interaction Layer
  • Ambient Wake
  • Overlay UI — GTK, Wayland layer-shell
  • Window & Session Control
  • Gesture Handlers — camera and keyboard
System Layer
  • IPC Core
  • Event Bus
  • Memory / State Store
  • Runtime for Macro Mapping
Plugins
  • Context Awareness
  • Language Model
  • Gesture Handler
  • Voice Model
  • Character-Render Engine (Wayland / Windows / Mac)
  • Vision Model
  • Macro Controller
